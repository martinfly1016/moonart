## Emoji List / Mobile Draft Sheet Work

The user reported that the mobile copy draft composer on:

- `/emoji-list/person-role/`

still did not match the existing text-tool pages.

Key issue:

- The emoji-list page had a custom mobile draft implementation.
- Existing text-tool pages had a more complete pattern:
  - top sheet bar
  - arrow state
  - `開く` / `閉じる`
  - fixed copy button
  - draft body below the bar
  - `0/160文字` meta
  - hint text
  - recent history section

Decision:

- Stop copying the pattern manually.
- Extract the mobile draft sheet into a shared JS component so future similar pages do not drift.

Implemented:

- Added shared component:
  - `assets/js/mobile-draft-sheet.js`
- Refactored:
  - `assets/js/text-tools.js`
  - `assets/js/emoji-list.js`
- Updated styles:
  - `assets/css/text-tools.css`
  - `assets/css/emoji-list.css`
- Updated text-tool HTML pages to load:
  - `/assets/js/mobile-draft-sheet.js`
  - before `/assets/js/text-tools.js`
- Updated `/emoji-list/person-role/` to load the shared component and removed its custom static mobile bar.

The shared component now owns:

- wrapping composer content into `.composer-body`
- moving draft textarea/actions into `.draft-editor`
- inserting `.composer-sheet-bar`
- managing:
  - `composer-open`
  - `has-mobile-composer`
  - arrow icon `⌃` / `⌄`
  - action text `開く` / `閉じる`
  - sheet copy button disabled state
  - sheet count
  - draft meta

Emoji-list-specific additions:

- `maxlength="160"`
- `data-draft-meta`
- `data-recent`
- recent history rendering
- clear history button
- copy button flash text
- draft-length enforcement
